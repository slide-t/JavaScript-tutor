Lesson 6 – Comparison Operators

🧠 What Are Comparison Operators?

Comparison operators let you check if one value is equal to, greater than, or less than another.

📌 The Most Common Operators

==    equal (loose check – allows type coercion)
===   strict equal (checks value + type)
!=    not equal (loose)
!==   strictly not equal
>     greater than
<     less than
>=    greater than or equal to
<=    less than or equal to

🔍 Examples

5 == "5"      // true (JS converts string to number)
5 === "5"     // false (types are different)
7 !== 7       // false (same value and type)
10 != "10"    // false (they’re equal loosely)
3 >= 2        // true
4 <= 4        // true
⚠️ Use === and !== for most comparisons. Avoid == unless you understand coercion!

👨‍💻 Real-World Use: Password Checker

let password = prompt("Enter your password");

if (password === "secret123") {
  alert("Access granted!");
} else {
  alert("Wrong password!");
}
✅ Use === to avoid bugs when checking user input, database values, or stored session data.

🧪 Practice

// What will this return?
console.log(2 == "2");     // ?
console.log(2 === "2");    // ?
console.log(3 > 2);        // ?
console.log(5 <= 5);       // ?
console.log("10" !== 10);  // ?
✅ Show Answers
  • 2 == "2" → true (coerced)
  • 2 === "2" → false (diff type)
  • 3 > 2 → true
  • 5 <= 5 → true
  • "10" !== 10 → true

🧠 Summary

  • == is loose – avoid in most cases
  • === is strict – use it often!
  • Use these operators in if statements to control logic
